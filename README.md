# 租清：房租水电管理微信小程序

一套可直接导入微信开发者工具运行的原生小程序，覆盖房东与租客之间的房间、抄表、计费、账单、收款、月结和退租闭环。当前已接入 Supabase 云同步适配层：未填写配置时保持纯本地模式，填写公开密钥并执行建表 SQL 后，启动时自动拉取、写操作自动上传。

## 直接运行

1. 打开微信开发者工具，选择“导入项目”。
2. 选择本目录，AppID 可先使用测试号。
3. 编译后先选择“房东”或“租客”身份；两种身份会进入独立页面与操作权限。
4. 浏览器直接打开 `index.html`，可体验独立的 iOS 与 Android HTML 预览。

## 已实现

- 微信快捷登录与本地体验入口；登录时明确选择房东或租客身份。
- 房东端与租客端路由隔离；租客只能查看本人账单，无法进入房间管理、抄表、收款和月结页面。
- 房间与租客绑定；房间页顶部选择房产，下方用独立卡片展示该房产的房间、入住状态和月租。
- 独立新增房间页面；一次填写小区或楼栋、详细地址、统一月租、押金及水电单价，可按起始楼层、楼层数量和每层房间数自动生成房间号，也可通过换行或逗号手动批量添加。
- 每个房间保存后仍可独立修改位置、房间号与收费标准，不会联动覆盖同批次的其他房间。
- 房东操作指南；从建房、邀请入住、抄表到出账收款均可一步直达。
- 水电抄表直接放在对应房间详情页；首次补填一次上期读数，后续自动带入，只需录入本期读数，保存后按差值生成或更新账单。
- 房租按月结算；水电费按“本月读数 − 上月读数 × 单价”计算。
- 每间房按入住日生成一个月租期和收租日；账单区分本月与历史记录，并继续按收租日分组，提醒时间不控制账单生成。
- 确认收款后自动为该房间创建下一月草稿账单，并结转水电表数；新一期抄表完成后转为待收。
- 房东月结统计和按位置收款率。
- 租客账单明细与本地模拟支付回写。
- 退租押金、扣款、退款和房间空置恢复。
- JSON 本地备份、导入恢复和可替换后端适配层。

## 接入 Supabase

详细配置步骤见 `supabase/README.md`。当前使用带 RLS 的 `app_states` JSONB 快照保持现有业务逻辑和离线可用性；数据量增大或开放多房东协作时，可逐步拆分为：

- `profiles`：微信用户与角色。
- `room_locations`、`rooms`：房间位置分组和房间。
- `tenancies`：租客绑定、租期与押金。
- `meter_readings`：按房间和月份唯一。
- `bills`、`payments`：账单与支付流水。
- `message_jobs`：微信订阅消息待发送队列。
- `checkouts`：退租结算快照。

自动发送微信账单需要在服务端持有微信凭证并执行订阅消息任务。当前前端已经实现授权入口与待发送队列语义；配置模板 ID、云函数或 Supabase Edge Function 后即可接通。

## 关键文件

- `data/store.js`：本地数据、计费闭环和备份恢复。
- `utils/billing.js`：计费纯函数。
- `services/backend.js`：后端替换边界。
- `services/notification.js`：微信订阅消息入口。
- `utils/auth.js`：登录会话、身份守卫与退出分流。
- `pages/login/`：房东/租客身份选择与登录入口。
- `pages/room/form/`：房东批量新增房间与统一收费标准配置。
- `pages/guide/`：房东完整操作指南。
- `index.html`：HTML 预览入口。
- `preview/shared.css`、`preview/shared.js`：跨平台预览共享系统。

## 租客入住资料邀请

- 房东从房间列表打开某个房间，可查看每位住户的姓名、完整手机号和完整身份证号。
- 房东点击“邀请租客填写”，生成绑定到该房间的微信分享路径；邀请只能以租客身份打开。
- 房东先在房间资料页选择入住日期，再发送邀请；租客不能填写或修改入住日期。
- 租客选择 1–6 人入住后，系统生成同等数量的资料卡；每人必须填写姓名和身份证，手机号允许留空，但整份资料至少保留一个有效手机号。
- 房东拥有本房间住户档案的最高管理权限，可逐人修改；退租统一通过退租结算处理。
- 完整证件资料仅向关联房间的房东和本人开放。接入 Supabase 时应配置行级权限、操作审计和敏感字段保护。
- HTML 预览入口已增加 iOS/Android 房间资料页与租客入住资料表单，可从 `index.html` 直接打开。
